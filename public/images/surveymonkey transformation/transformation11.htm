Narrowing down the options for a path forward

60 px

Build with AI

Utilize our new chat GPT feature for an enhanced survey authoring experience. We will gradually add features to match the existing experience.

Pros:
• Modern, lightweight experience
• AI Built in from the ground up 

Cons:
• Time to parity could be extensive
• It would create a bifurcated experience

Focus on the 80%

Create a new user experience focusing on the top 80% utilized features. Gradually add remaining features for full parity.


Pros:
• Lightweight modern experience
• Unhindered by the current code base 

Cons:
• It could take years to complete and we might Lose paid customers along the way

Basic experience

Build a new experience for our basic/free users and have a switch that would take upgraded users to our legacy product.

Pros:
• Fast to build
• Unhindered by the current code base 

Cons:
• Paid users would be downgraded to a poor experience when they upgrade
• It could take years to complete

The path forward

After consulting with leadership, we worked with the engineering team for another week to explore better options.  Realizing that our goal was to migrate from Python to React, I shared a hybrid concept that showed React components injected into the Python UI.   Teaming up with a Staff architect, we devised a method to integrate React components within the existing Python code.

This approach enables us to provide incremental value efficiently and mitigate bigger risks.
